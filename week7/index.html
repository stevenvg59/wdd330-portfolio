Functions are first-class objects so they can have their own properties and methods. Length(), call() and apply() are very useful methods that we can implement them so that helps the functions are not tied to specific objects and are more flexible. Immediately Invoked Function Expression (IIFE) is a function that is invoked as soon as it's defined. Here is the syntax:

(function(){
console.log(`Hello ${temp}`);
})();
<< 'Hello World'

One advantage of using IIFE is that we can have temporary variables (variables that are only used in part of the program and then disappear) and do not overfill the code with lots of variable names. We can also isolate blocks of code and keep variables with the same name but without interfering with each other. For example:

function(){
console.log(`Hello ${temp}`);
}();

<< 'Hello World'

(function() {
// block A
const name = 'Block A';
console.log(`Hello from ${name}`);
}());
(function() {
// block B
const name = 'Block B';
console.log(`Hello from ${name}`);
}());
<< Hello from Block A
<< Hello from Block B

Both use the variable "name" but they are independent.

I learned that we can use functions within functions and that variables used in outer variables (variables declared in outer functions) can be also used within the inner functions. This is a key example of using closures.

function outer() {
const outside = 'Outside!';

function inner() {
const inside = 'Inside!';
console.log(outside);
console.log(inside);
}
console.log(outside);
inner();
}
The output will be:
outer()
<< Outside!Inside!Outside!
